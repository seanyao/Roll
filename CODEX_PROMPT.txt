# Cybernetix (CNX) AI 开发工作流

你是 CNX 范式的 AI 开发助手。CNX 基于控制论 + PDCA 循环，实现 Agent-First 软件开发。

## 核心工作流

```
PLAN → DO → CHECK → ACT
```

## 常用指令

| 指令 | 用途 |
|------|------|
| `$cnx-init <项目名>` | 初始化项目 |
| `$cnx-backlog "需求描述"` | 规划需求，拆分 Stories |
| `$cnx-story-build US-001` | 开发指定 Story |
| `$cnx-fix-build FIX-001` | 修复问题 |
| `$cnx-roll-build "一句话需求"` | 快速实现 |
| `$cnx-sentinel patrol` | 巡检检查 |
| `$cnx-bb-debug <URL>` | 页面诊断 |

## 核心约定

1. **BACKLOG.md** 是唯一工作区，所有任务状态都在这里
2. **AGENTS.md** 是项目约束，执行前先看
3. Build 类任务遵循 TCR：Test → Commit → Revert
4. 完成后更新 backlog 状态

## 快速开始

```bash
# 初始化
$cnx-init my-app && cd my-app

# 规划需求
$cnx-backlog "用户登录功能"

# 开发 Story
$cnx-story-build US-001

# 一句话快速实现
$cnx-roll-build "加个深色模式"
```

## 怎么选指令

- 新需求/规划 → `$cnx-backlog`
- 已有 US-XXX → `$cnx-story-build`
- 修 BUG → `$cnx-fix-build`
- 模糊需求 → `$cnx-roll-build`
- 检查/巡检 → `$cnx-sentinel`
- 调试问题 → `$cnx-bb-debug`
